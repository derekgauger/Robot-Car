◆ taskMethod()

void se3910RPi::CollisionSensor::taskMethod ( )
virtual

This is the task method. The task method will be invoked periodically every taskPeriod units of time.

1.0 Start by reading the distance from the distance sensor.

2.0 If the distance is >= to the mad, reset the violationCount to be 0.

3.0 Else the distance must be less than the mad.

3.1 Increment the violation count number.

3.2 If the violation count is exactly 2

3.2.1 Send a stop command to stop the robot from moving.

3.2.2 Send a command to cause the horn to sound.

3.3 If the violation count is 3, send a second event to cause the horn to sound. There's a reason why this is necessary. Looking at the sequence of events between the robot controller and this, can you see why it is necessary?

int se3910RPi::CollisionSensor::madViolationCount =0

This variable keeps track of the number of times a reading less than the minimumacceptabledistance has been recorded. If 2, it will send a stop command. It will continue counting upwards until a value less that the mad occurs at which point it will reset to 0.
