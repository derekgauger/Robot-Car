void Horn::taskMethod ( )
virtual

This is the task method for the class.

1.0 Start by determining if there are any events to be processed off of the queue. In this case, do this as a while loop so that all of them are processed before we get further into the loop. We use a while loop here because this task is pretty slow, so we may have a few events that come in for us to process.

1.1 There is at least one item in the queue. So dequeue the first item.

1.2 If the event is an order to silence the horn, invoke the silenceHorn method.

1.2.1 Silence the horn.

1.3 Else if the command is a sound command, force the horn to sound.

1.3.1 Invoke the soundHorn method.

1.4 Else if the command is a Pulse command, set up the parameters for a pulsed horn.

1.4.1 Extract from the bits the period and the on time for the horn. The lowest 12 bits indicate the period for the horn sound in ms. The next 12 bits indicate the time that the horn is to sound in ms.

1.4.2 Invoke the pulse horn method passing in the parameters extracted from the message, converting them to microseconds.

2.0 Determine if the count is -1. If it is, do nothing, as no sound should be made.

2.0.1 If horn count is -1, set the horn GPIO pin to be low. Should be silent.

2.1.1 otherwise, If the horn count is 0,

2.1.1.1 the horn should be sounding, so turn it on by setting the GPIO pin high.

2.1.1.2 Increment the hornCount by the HORN_TASK_RATE constant, indicating that the horn has now sounded this much longer.

2.1.1.3 Set the horn count to be the remainder of the horn count divided by the repetition time.

2.2.1 Otherwise, we are in the middle. The horn has been sounded, we need to determine whether to keep it sounding or make it quiet.

2.2.1.1 Determine if the hornCOunt is less than or equal to the legnth.

2.2.1.1.1 We are in the time period between 0 and length. The horn should be sounding. Therefore, set the horn pin to be high.

2.2.1.2.1 Otherwise, the horn count is > the length. It should be silent right now. Set the pin low.

2.2.1.2 Increment the horn count by the HORN_TASK_PERIOD.

2.2.1.3 Set the hornCount to be the remainder of the hornCount divided by the repetitionTime.
