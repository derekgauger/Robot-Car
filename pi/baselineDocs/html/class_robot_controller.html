int RobotController::processMotionControlCommand ( int  value)
protected

This method will process a command that is related to motion control.

Parameters
valueThis is the command that was received.
Returns
This is the command that was last processed. Under normal circumstances, it is exactly the value as the value passed in.

1.0 Based on the command passed in, set the left and right motor directions so that the robot steers accordingly. This is quite a bit of logic here.

1.0.1 If the direction is forward, set all 4 motors to run forward.

Then silence the horn.

1.2 If the direction is left, set the motors on the left side of the robot to run in reverse and on the right side to run forward.

Then silence the horn.

1.3 If the direction is right, set the motors on the right side to run in reverse and the motors on the left side to run forward.

Then silence the horn.

1.4 If the direction is backward, set the motors to run in reverse.

Then beep the horn with a period of 600ms whereby it sounds for 300ms.

1.5 If the direction is forward left, set the left motors to be turned off and the right motors to run forward.

Then silence the horn.

1.6 If the direction is forward right, set the right motors to be turned off and the left motors to run forward.

Then silence the horn.

1.7 If the direction is backward left, set the left motors to be turned off and the right motors to run backward.

Then beep the horn with a period of 600ms whereby it sounds for 300ms.

1.8 If the direction is backward right, set the right motors to be turned off and the left motors to run backward.

Then beep the horn with a period of 600ms whereby it sounds for 300ms.

1.9 If the direction is stop, set all motors to stop.

Then silence the horn.

◆ processSpeedControlCommand()

void RobotController::processSpeedControlCommand ( int  value)
protected

This method will process a command that is related to speed control. It will first verify that the value passed in is in range and then will update the motors accordingly based on the new parameter passed in.

Parameters
valueThis is the command that was received. It is expected to be a value between 0 and 1000 (inclusive).

This method will process a command that is related to speed control.

Parameters
valueThis is the command that was received after removing any additional control bits. The value should be between 0 and 1000, representing 0.0% duty cycle to 100.0% duty cycle in units of 0.1%.
  1. Start by setting the current speed to the value passed in.
  2. Now that we have this, set up the speed for the left and right motors based upon the current speed and the steering value. If the steering value is 0, the speed of the motors is just the current speed. However, if the value is > or < 0, the value of the motors will be scaled. For example, a value of -50 will cause the robot to turn slightly to the left. The motors on the right side will run at the current speed but the motors on the left will run at 50% of the current speed. A value of 50 would do the opposite. A value of -100 or 100 will cause one set of motors to be completely stopped.
  3. Set the motor speeds for all 4 motors controlled by the robot based upon what was calculated in the previous step.

This method will process a command that is related to steering control. It will first verify that the value passed in is in range and then will update the motors accordingly based on the new parameter passed in.

Parameters
valueThis is the command that was received. It is expected to be a value between -100 and 100 (inclusive).

This method will process the incoming steering command.

Parameters
valueThis is the steering command that is passed in.

1.0 Check to make sure that the value is in the range of -100 to 100.

1.0.1 If in range, start by setting the current steering value to the value passed in.

1.0.2 Now that we have this, set up the speed for the left and right motors based upon the current speed and the steering value. If the steering value is 0, the speed of the motors is just the current speed. However, if the value is > or < 0, the value of the motors will be scaled. For example, a value of -50 will cause the robot to turn slightly to the left. The motors on the right side will run at the current speed but the motors on the left will run at 50% of the current speed. A value of 50 would do the opposite. A value of -100 or 100 will cause one set of motors to be completely stopped.

1.0.3 Update the motor speeds based upon the value calculated for the left and right speeds.

1.1 Else if the value is out of range, do nothing to the value passed in.

◆ run()

void RobotController::run ( )
virtual

This is the run method for the class. It contains the code that is to run periodically on the given thread.

This is the run method. It contains the logic for this thread which deals with running the game.

1.0 Start each of the 4 motors up.

2.0 While the thread is still running

2.1.0 Obtain the next command from the queue. We will block here if there is nothing.

2.2.0 Process the incoming command accordingly.

2.2.1.0 If the command is a motor direction command, call the process motion control command method with the parameter.

2.2.2.0 If the command is a speed control command, call the method to process it.

2.2.3.0 If the method is a steering command, call the method to process it.

3.0 If we ever exit out of the loop, stop all 4 of the tasks we started at the beginning of the run method.

◆ currentSteering

int RobotController::currentSteering = 0
protected

This is a value for the current steering control. This value can range between -100 and +100, with 0 being straight ahead. It is used to adjust the speed of the motors relative to each other on the left and right side of the robot. A value of -100 will turn the robot to the left, causing the left wheels to be completely stopped; a value of 100 will turn the robot to the right, causing the right wheels to be completely stopped.
