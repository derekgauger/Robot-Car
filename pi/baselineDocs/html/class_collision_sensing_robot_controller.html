This method will process a command that is related to speed control. It will first verify that the value passed in is in range and then will update the motors accordingly based on the new parameter passed in by invoking the parent method. It will then update the mad stopping distance for the collision sensor. You will need to figure out what the mad should be based upon robot stopping distances and the PWM value. void CollisionSensingRobotController::run ( )
virtual

This is the run method for the class. It contains the code that is to run periodically on the given thread.

This is the run method. It contains the logic for this thread which deals with running the robot.

1.0 Start each of the 4 motors up.

2.0 While the thread is still running

2.1 Obtain the next command from the queue. We will block here if there is nothing.

2.2 Process the incoming command accordingly.

2.2.1 If the command is a motor direction command, call the process motion control command method with the parameter.

2.2.2 If the command is a speed control command, call the method to process it.

2.2.3 If the method is a steering command, call the method to process it.

3.0 If we ever exit out of the loop, stop all 4 of the tasks we started at the beginning of the run method.
